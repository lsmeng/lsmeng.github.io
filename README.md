# Lingsen Meng — Academic Homepage

Static personal website (plain HTML/CSS, **no build step**, **no Google Fonts** — so it
loads fast everywhere, including mainland China). Edit the `.html` files directly; to
update the publication list, edit `gen_pubs.py` and run `python3 gen_pubs.py`.

## Files
- `index.html` — home (bio, news, research highlights, selected papers, contact)
- `research.html` — research topics
- `publications.html` — full list (generated by `gen_pubs.py`)
- `group.html` — members & alumni
- `style.css` — all styling
- `assets/profile.jpg` — **add your photo here** (then see "Add your photo" below)
- `cv.pdf` — **add your CV PDF here** (optional; the "CV" button links to it)

## Add your photo
1. Put a portrait at `assets/profile.jpg` (≈ 600×750 px works well).
2. In `index.html`, replace the placeholder `<div class="photo-ph">…</div>` with:
   ```html
   <img class="photo" src="assets/profile.jpg" alt="Lingsen Meng">
   ```

## Preview locally
```bash
python3 -m http.server 8000
# open http://localhost:8000
```

## Deploy — GitHub + Cloudflare Pages (recommended, fast in China)

### 1. Put it on GitHub
```bash
git init
git add .
git commit -m "Initial site"
# create an empty repo on github.com first, then:
git branch -M main
git remote add origin https://github.com/<you>/<repo>.git
git push -u origin main
```

### 2. Connect Cloudflare Pages
1. Sign in at https://dash.cloudflare.com → **Workers & Pages** → **Create** → **Pages** → **Connect to Git**.
2. Pick this repo.
3. Build settings: **Framework preset = None**, **Build command = (leave empty)**,
   **Build output directory = `/`** (the repo root). It's a static site, so there is nothing to build.
4. **Save and Deploy**. You'll get a free `https://<project>.pages.dev` URL with global CDN
   (generally faster and more reliable from China than `github.io`).

Every `git push` to `main` re-deploys automatically.

### (Optional) Custom domain
In the Pages project → **Custom domains** → add e.g. `lingsenmeng.com`. If the domain's DNS is on
Cloudflare, this is one click and gives you HTTPS automatically.

## Alternative: GitHub Pages only
Repo **Settings → Pages → Source: Deploy from branch → `main` / root**. Site appears at
`https://<you>.github.io/<repo>/`. Works, but can be slow/intermittent in mainland China —
Cloudflare Pages is the better default.

## (Optional) Gitee mirror for China
For the most reliable access inside China, also push to [Gitee](https://gitee.com) and enable
Gitee Pages as a China-only entry point. Requires real-name verification.
